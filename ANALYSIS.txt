This algorithm checks through each edge in the matrix and evaluates whether there is an existing edge that has a smaller distance than the current distance. Essentially, instead of using nested loops in this function it uses recursion to call the next iteration. 

For the runtime, there are three things to consider combining. We need to pair every intermediate node, k, with every source node, i, and every destination node, j. This means that for each of this we need to run n times, where n is the number of nodes in the graph. 
Leaving us with a theoretical runtime of O(n^3).
